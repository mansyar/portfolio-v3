## Tubular BEXUS Onboard Software

Onboard software developed for the TUBULAR BEXUS 26/27 experiment — a student-led research project launched as part of the REXUS/BEXUS programme, which provides stratospheric balloon and sounding rocket platforms for university experiments.

### Background

The REXUS/BEXUS programme (Rocket and Balloon Experiments for University Students) is a collaboration between the German Aerospace Center (DLR), the Swedish National Space Agency (SNSA), and the European Space Agency (ESA). TUBULAR's experiment was designed for stratospheric flight to conduct scientific measurements using onboard sensors and data logging.

### Key Responsibilities

- **Onboard Data Acquisition**: Capture and log sensor telemetry during stratospheric flight
- **Telemetry Handling**: Process and transmit experimental data through the rocket's telemetry downlink
- **Flight Safety**: Implement watchdog timers and fault-safe states for autonomous operation
- **Payload Integration**: Interface with the BEXUS service systems (power, comms, GPS) per the experiment interface specification
- **Post-Flight Analysis**: Store experiment data in onboard flash memory for recovery after landing

### Tech Stack

Written in C for embedded platforms, with direct hardware register access for sensor interfaces and telemetry UART communication. Development followed the REXUS/BEXUS experiment lifecycle: PDR (Preliminary Design Review), CDR (Critical Design Review), integration testing, and launch campaign.

### Architecture

The software follows a simple super-loop architecture suitable for resource-constrained embedded systems. Initialization routines configure sensors and communication interfaces, then the main loop cycles through data acquisition, telemetry packetization, and health monitoring at deterministic intervals. Watchdog timers provide automatic reset on fault conditions, and a write-once flash storage scheme ensures data integrity through landing impact.
